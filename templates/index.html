{% if data.mode=="secure" %}

Secure Communication:

Diffie-Hellman enables two users to establish a shared secret over an insecure channel.

Core Idea:

Instead of sending the key, both compute it independently.

Formula:

K = gab mod p

Why Secure?

  • Attacker sees A = g^a and B = g^b
  • Cannot compute a or b
  • Requires solving Discrete Log → very hard

Security Achieved:

  • Confidentiality ✔
  • Integrity (partial) ✔
    • {% elif data.mode=="attack" %}

    MITM Attack:

    An attacker secretly intercepts communication between two parties.

    How it works:

    • Eve intercepts A and B
    • Sends fake values (E1, E2)
    • Creates two separate keys

    Why possible?

    • No authentication mechanism
    • Users trust received public values blindly

    Impact:

    • Confidentiality ❌ (Eve reads data)
    • Integrity ❌ (Eve modifies data)
    • Trust ❌ (communication compromised)
    {% elif data.mode=="auth" %}

    Authenticated Diffie-Hellman:

    Enhances DH by verifying identities of participants.

    How it works:

    • Public keys are digitally signed
    • Receiver verifies using sender’s public key

    Algorithm concept:

    Signature = Sign(A, Private Key)

    Verify(Signature, Public Key) ✔

    Why secure?

    • Eve cannot forge valid signatures
    • Fake keys are rejected

    Security Achieved:

  • Confidentiality ✔
  • Integrity ✔
  • Authentication ✔
